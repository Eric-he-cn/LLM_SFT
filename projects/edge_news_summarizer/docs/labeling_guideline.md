# 数据标注指南

## 1. 结构化摘要格式规范

每条新闻的结构化摘要必须包含以下 6 个字段，格式严格如下：

```
【一句话摘要】
（用一句话概括新闻核心内容，不超过 50 字）

【核心要点】
1. （第一个关键信息点）
2. （第二个关键信息点）
3. （第三个关键信息点）

【事件类别】
（类别标签）

【主要主体】
（主要涉及的人物、机构或组织）

【时间信息】
（关键时间节点，若无则填"未提及"）

【潜在影响】
（事件的潜在影响，不超过 80 字）
```

---

## 2. 字段说明与标注细则

### 2.1 【一句话摘要】

- **长度**：不超过 50 字（中文）
- **内容**：必须包含主体 + 事件 + 结果三要素
- **风格**：客观、简洁、中性
- **禁忌**：不得包含主观评价词（如"震惊"、"令人担忧"等）

**✅ 好的例子**
> 苹果公司于 2024 年 9 月发布 iPhone 16 系列，新增 AI 功能并提升相机性能。

**❌ 不好的例子**
> 苹果又出了一款令人兴奋的手机！

---

### 2.2 【核心要点】

- **数量**：最少 3 条，最多 5 条
- **格式**：必须使用阿拉伯数字编号（1. 2. 3.）
- **内容**：每条聚焦一个独立信息点
- **长度**：每条不超过 40 字

**✅ 好的例子**
```
1. iPhone 16 系列共四款机型，起售价 799 美元
2. 全系搭载苹果 A18 芯片，支持 Apple Intelligence
3. 主摄像头升级至融合相机系统，支持 4K 120fps 录制
4. 中国大陆首批不支持 Apple Intelligence 功能
```

**❌ 不好的例子**
```
1. 苹果发布了手机
2. 有很多新功能
3. 价格不变
```

---

### 2.3 【事件类别】

必须从以下类别中**精确选择一个**（注意：不能自造类别）：

| 类别 | 适用场景 |
|------|----------|
| 科技 | 科技产品、互联网、人工智能、软件 |
| 财经 | 金融市场、企业财报、经济政策、投资 |
| 政治 | 选举、政府政策、外交、立法 |
| 社会 | 民生、教育、医疗、公共事务 |
| 体育 | 竞技体育赛事、运动员、体育政策 |
| 文化 | 电影、音乐、艺术、出版、游戏 |
| 国际 | 国际关系、国际组织、跨国事件 |
| 军事 | 军事行动、武器装备、国防政策 |
| 环境 | 气候变化、自然灾害、生态保护 |
| 健康 | 医学研究、疾病、公共卫生 |

> ⚠️ 如果新闻涉及多个类别，选择**最主要**的一个。例如：AI 医疗诊断属于"科技"或"健康"，根据文章侧重点选择。

---

### 2.4 【主要主体】

- **格式**：多个主体用顿号（、）分隔
- **内容**：人名/机构名/组织名，优先写全称
- **数量**：1-4 个

**✅ 好的例子**
> 苹果公司、蒂姆·库克

**❌ 不好的例子**
> 一家科技巨头

---

### 2.5 【时间信息】

- 若原文有明确时间：提取最关键的 1-2 个时间节点（年月日）
- 若原文无明确时间：填写"未提及"
- 不得填写"最近"、"近日"等模糊表达

**✅ 好的例子**
> 2024年9月9日发布，2024年9月20日开售

**❌ 不好的例子**
> 最近发布

---

### 2.6 【潜在影响】

- **长度**：不超过 80 字
- **内容**：分析此事件可能的后续影响（对行业、社会、政策等）
- **风格**：客观分析，避免情绪化表达
- **要求**：至少提及一个具体的影响方向

---

## 3. 人工评测评分标准

用于 `test_manual_eval.json` 的 100 条人工评测样本。

### 评分项（每项 0 或 1 分，满分 5 分）

| 评分项 | 说明 |
|--------|------|
| 关键主体是否提及 | 摘要中是否正确识别了新闻的核心人物/机构 |
| 核心事件是否准确 | 一句话摘要是否正确描述了核心事件 |
| 关键时间是否提及 | 若原文有明确时间，摘要是否正确提取（无时间则自动给分） |
| 是否有事实错误 | 摘要中是否包含与原文矛盾的陈述（0=有错误，1=无错误） |
| 是否有明显幻觉 | 摘要中是否包含原文未提及的内容（0=有幻觉，1=无幻觉） |

### 评分示例

```json
{
  "id": "news_abc123",
  "score": {
    "key_entity": 1,
    "core_event": 1,
    "time_info": 1,
    "no_factual_error": 1,
    "no_hallucination": 0
  },
  "total": 4,
  "notes": "潜在影响部分提到了'将超越谷歌'，原文并未提及，疑似幻觉。"
}
```

---

## 4. 常见标注错误

1. **缺少编号**：要点写成 `-` 或 `·` 而非 `1. 2. 3.`
2. **类别自创**：使用"商业"、"经济"等白名单外的类别
3. **一句话摘要过长**：超过 50 字
4. **时间模糊**：使用"近期"、"今年"等相对表达
5. **主体不具体**：写"一家公司"而非具体公司名
